If specified, terraform workspaces are utilized with the specified name by executing the following commands. If the workspace does not exist, it will be created.

terraform workspace new "name"

terraform workspace select "name"

If specified along with "Destroy on Build Completion", the workspace will also be deleted along with the managed resources.

Workspaces can be used with supporting remote backends. If not, the Jenkins workspace is used and must be properly protected from deletion and sharing to allow multiple Terraform workspaces to coexist.

Use blank to disable workspaces selection, i.e. terraform workspace will not be called.

Use "default" select the default terraform workspace.

Workspaces allow multiple users, environments, configurations to share the same configuration while storing unique state per workspace.

For more information, see: Terraform Workspaces.
